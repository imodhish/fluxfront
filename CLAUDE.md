# FLUXFRONT ‚Äî Project Context

Real-time fluid-defense strategy game (Creeper World 3 inspired). Vanilla JS + Canvas 2D, zero dependencies. Entry point `index.html` + ES modules under `src/`. `fluxfront.html` is the original single-file prototype ‚Äî a historical snapshot only (it still runs, but the modular game has since diverged: bigger 128√ó80 map, retuned `DIFFS`, full presentation overhaul). Serve over HTTP (e.g. `python -m http.server`) ‚Äî ES modules don't load from `file://`.

## Architecture (`src/` ES modules)

- **constants.js** ‚Äî game constants, `TYPES` (structure stats), `DIFFS`, seeded RNG (`mulberry32`), small utils (`idx`, `clamp`, `dist`, ‚Ä¶).
- **state.js** ‚Äî shared mutable state (`S`, `selBuild`/`sel`/`delMode`, `paused`/`speed`/`muted`, `hover`, `shake`, canvas refs, `el`, camera `cam{x,y,z}`, `dpr`, `uiScale`). Imports are read-only live bindings, so cross-module reassignment goes through the exported setters (`setS`, `setShake`, ‚Ä¶); property writes on `S`/`hover`/`el`/`cam` are direct.
- **world.js** ‚Äî value-noise terrain (`genTerrain`: 12 levels via plateau-biased terracing ‚Äî ~80% of each height band snaps flat ‚Äî plus two 3√ó3 majority-filter passes, so maps are dominated by large buildable plateaus; ~20% of cells anchor a flat 3√ó3), emitter placement with basin carving, terrain prerender to an offscreen canvas, `newGame()`. The bake is **supersampled** (`SS` in world.js: 2√ó on small maps, 1.5√ó on L, 1√ó on XL/HUGE for bake cost) into a BW√óBH (=W¬∑SS) offscreen, then drawn back down to W√óH with smoothing ‚Äî true SSAA, so cliff edges/contours are crisp and zoom-in stays sharp. All bake coords are in bake pixels: `BW`/`BH` dims, `CPX`=CELL¬∑SS px/cell, noise frequencies divide by SS (world-constant feature size), `LV` slope scales √óSS. The pipeline (~1 s full map, once per game): coarse 4 px noise fields ‚Üí domain-warped, terrace-sharpened height field (`rebuildHm`: a coarse low-freq warp PLUS a finer high-frequency per-pixel wiggle, clamped to the padded grid, so cliff edges follow eroded organic curves instead of the axis-aligned cell-grid staircase ‚Äî premium even at max zoom) ‚Üí half-res ray-marched sun shadows (`rebuildShadows`) ‚Üí `shadeRegion` (material splatting: ridged cliff rock with strata/rust/erosion streaks, ochre dust flats, teal mineral stains, mud-cracked wet basins; Lambertian warm-sun/cool-ambient light **lit through a micro-bump normal** (gradient of a per-bake-px fBm detail field `bk.bump`, so flats/rock catch the raking light ‚Äî material classification still uses the clean terrain normal), Blinn specular via squaring chain, 4-tap AO, smooth topographic contours (a thin dark line at each plateau boundary, driven by the continuous height field so it curves organically rather than stepping along cells), **sunward crest catch-light**, then a **tonemap/grade**: saturation + warm-highlight/cool-shadow + aerial-perspective height tint + gentle contrast). Decor pass draws lit **boulders with SE cast shadows**, **scree at cliff bases**, **wind ripples on flats**, basin stains (density ‚àù map area, separate RNG). All intermediates are cached per seed in module state (`bk`); `retileTerrain` repaints only the Terraformer-affected pixels (margins √óSS). At SS=1 every constant is identical to the pre-supersampling baker. The render draws `terCan` (BW√óBH) scaled to W√óH with `imageSmoothingQuality='high'`. Inner loops use inlined indexing, not closures ‚Äî keep it that way. Seeded decor uses a *separate* RNG ‚Äî nothing here may consume `st.rng`, which drives the sim.
- **sim.js** ‚Äî `simCreeper` (Flux fluid), `tickEmitters`, combat (`tickBuildings`: cannon/mortar/nullifier logic, `blast`, `tickShells`), placement (`canPlace(st,type,cx,cy,ignoreId?)` ‚Äî the whole footprint must sit on one terrain level ('Uneven ground'; core needs a flat 3√ó3, sz-2 structures a flat 2√ó2); **no network-link requirement**: off-grid placements wait as dormant ghosts until the network reaches them. The **Core may deploy anywhere flat & Flux-free, even right beside an Emitter** ‚Äî `canPlace` short-circuits `type==='core'` to `{ok:true}` past the footprint check; other structures still keep a 1-cell Emitter buffer. The sim is **frozen during the `placeCore` phase** (`tick()` early-returns after `tickFX` while `phase==='placeCore' && !coreDown`), so no Flux flows and no clock runs until the Core lands ‚Äî an orbital RECLAIM (`coreDown`) is the exception and keeps flooding), relocation (`startMove`/`MOVABLE` = core/cannon/mortar: destination validated with `ignoreId`, target cells claimed immediately, the structure flies at 55 px/s ‚Äî airborne it takes no Flux damage, can't fire, is off-network, and a flying core downs the whole network), `deconstruct` (blocked while moving), Terp painting (`terraPaint`/`applyTerraStep`), surges (`tickSurge`), erosion (`tickErosion`), relics (`tickRelics`), Power Zones (`updatePZ`), win/lose, FX, master `tick()`.
- **net.js** ‚Äî `recomputeNet` BFS, `pathTo`, `linkTargets`.
- **economy.js** ‚Äî `tickEnergy`, `dispatchPacket`, `tickPackets`, `deliver`.
- **render.js** ‚Äî canvas toast messages (`msg`/`drawMsgs`), all `draw*` render functions, `render()`. FX layer: lazily pre-rendered sprites (per-color radial glow cache, vignette, red danger vignette, scanlines), building spawn-in animation (`spawnMap` keyed by building id, render-only ‚Äî sim objects untouched), animated dash flow on network links, glows on emitters/core/packets/shells/shot impacts. Flux rendering: depth ramp azure‚Üíindigo with a bright rim on cells bordering dry land, animated surface waves, topographic depth bands (every ~1.25 depth), + additive second draw as bloom. Emitters are obsidian heptagons with molten cracks, rotating containment ring and pulsing core (cooled husks when dead). `drawCoverage` tints the union of collector+core link radii with a single translucent fill (drawn over terrain, under Flux). Structure bodies are baked once into 6√ó supersampled offscreen sprites (`blitSprite` + per-type `sprite*` painters ‚Äî icon-grade gradients/bevels/rivets/hazard stripes, riveted foundation plate with drop shadow included); animated parts draw live on top in `drawShape`: core's rotating ring + reactor heart + beacons, collector glow pulse, relay radar sweep + beacon blink, reactor torus/core pulse, battery charge segments, cannon turret (own sprite, rotated by `b.aim` with recoil + muzzle flash), mortar loading bore glow, nullifier prongs/orb/20-tick charge ring. Canvas gradients for dynamic parts are cached in `gradCache`. Every structure has a blinking status LED (green = connected, red = orphaned). Packets draw short motion trails along their current path segment. **Full-frame bloom** (`drawBloom`, called at the end of `render()` after the world+vignette but before the crisp UI text/frame): downscales the rendered frame to ¬º res (= cheap blur) into a cached `bloomCan`, multiplies by mid-grey to crush midtones (bright-pass), then composites it back additively (`lighter`, Œ± 0.85) upscaled ‚Äî so Flux, beams and explosions glow cinematically. Gated by `settings.bloom` (SETTINGS ‚Üí Bloom glow, persisted).
- **ui.js** ‚Äî HUD/overlay DOM wiring, pointer + keyboard input, `init()`, fixed-timestep `frame()` loop. Gesture layer: tap acts on pointerup (drag beyond ~5‚Äì9 px = camera pan, pinch = zoom, wheel = zoom); on touch, placement/recycle/relocation is two-tap (first tap arms a preview at `armedCx/Cy`/`armedB`, second tap on the same target commits). In build mode the primary pointer **paints** instead of panning (`tryPaint`): dragging chain-builds along the path ‚Äî backbones auto-space at `linkR*0.85`, others at `sz`; pinch still zooms. Relocation flow: clicking an already-selected movable structure sets `moveSrc` (state.js), next confirmed map click calls `startMove`; Esc/right-click or mode changes clear it. In **recycle mode** (X / btnDel) the primary drag draws a `marquee` box (state.js) ‚Üí `recycleBox` removes every enclosed built non-core structure on release; a **double-click** on a structure ‚Üí `recycleSameType` removes that type within 6 cells. In **move mode** (`moveMode`, G / btnMove) the box selects MOVABLE built structures into `moveGroup` (state.js); the next click `relocateGroup`s them all to that cell preserving formation around the group centroid (`startMove` per member, invalid targets skipped). `drawMarquee` tints red for recycle / blue for move; `drawMoveGroup` outlines the selection and shows a formation ghost at the cursor (blue ok / red blocked). Both modes are mutually exclusive and cleared by `cancelModes`. **Guided first mission**: the menu's FIRST MISSION button starts EASY on the smallest map with fixed seed `TUT_SEED` and `opts.tutorial:true` ‚Üí `S.tut`/`S.tutStep`. `tutCheck()` (called from `frame()`, render-side only ‚Äî never mutates sim arrays, so determinism holds) drives a `TUT_STEPS` state machine: each step shows the `#tutCard` coach overlay and advances when its `done(st)` predicate is met (deploy core ‚Üí collector ‚Üí relay ‚Üí cannon ‚Üí nullifier ‚Üí win). SKIP (`tutSkip`) or reaching the last step calls `endTutorial`. INFO steps wait for the NEXT button (`tutAdvance`); ACTION steps (those with a `done` predicate) auto-advance and turn NEXT into "skip step". **First-use structure tips**: the first time the player selects any build type, `showTip(key)` (from `selectBuild`) shows a one-shot `#tipCard` blurb from `STRUCTURE_TIPS`, then `markTipSeen` persists it (storage.js `DB.tips`); suppressed during the tutorial, gated by `settings.tips` (SETTINGS ‚Üí Structure tips, with a "replay all" `resetTips` button). **Persistent alert strip** (`#alerts`, `renderAlerts` in `updateHUD`): chips for redeploy-core / network-down / core-under-attack / surge / squall / spore-inbound / low-energy, rebuilt only when the set or a countdown second changes; the energy `#energyBar` gains `.low` (pulsing) when guns are starving. Build-button hover tooltips are enriched (`tipStats`: cost/size/hp + the relevant stat) and `onKey` ignores keypresses while an INPUT/TEXTAREA/SELECT is focused. A `?` key (or Shift-/) toggles the `#keys` keyboard cheat-sheet overlay. **Selected-structure panel** (`selPanel`): the `#info` panel renders hp/ammo/charge/build bars (`mbar`), a live stat line, a plain-language status line (disconnected / out-of-ammo / unpowered / stunned / operational), and recycle/relocate buttons wired by click-delegation on `#info`. **Save / resume in progress**: `buildSnapshot` captures seed+diff+mods+size+action-log+tick for plain (non-sandbox/tutorial/daily/challenge/replay) games; `autoSave` runs every 8 s, on opening the ‚ò∞ menu, and on `beforeunload` (storage.js `cont` slot, cleared by `startGame`/`showEnd`). The menu **CONTINUE SECTOR** button calls `resumeGame`, which restarts the deterministic game and silently fast-forwards (`pumpReplay`+`tick` with audio muted) to the saved tick, then hands back live control. An optional **FPS readout** (`#fps`, `drawFps`, SETTINGS ‚Üí Show FPS) measures frame rate. This is the entry module loaded by `index.html`; circular imports between modules are intentional and safe (function declarations, runtime-only calls).
- **audio.js** ‚Äî WebAudio sound design (lazy init on first pointer event): master gain ‚Üí compressor chain, sfx synthesized from oscillators + filtered noise, stereo-panned by world x (`sfx(kind, x?)`), `applyMute()` gates the master bus. Kinds include uiclick/error/alarm beyond the combat set.
- **music.js** ‚Äî procedural adaptive music: sub drone + A-minor pad progression (Am‚ÄìF‚ÄìC‚ÄìEm, 4 s/chord) + echoed arpeggio + pulse bass, on a 0.5 s lookahead scheduler. `setMusicIntensity(0..1)` (fed from `updateHUD`: Flux coverage + elapsed time) crossfades layers; deterministic internal LCG keeps it allocation-light. No audio assets.

`tools/verify-split.mjs` (run with `node`) runs a scripted seeded game on stubbed DOM and asserts: same-seed determinism across two fresh runs, sim invariants (finite/bounded Flux, energy ‚â§ cap, consistent building registry), and a full render smoke pass. Rerun it after any sim or render change. (It originally asserted bit-equivalence against `fluxfront.html`; that was retired when the map size diverged.)

## Core systems

- **Grid**: selectable map size ‚Äî `COLS`/`ROWS`/`W`/`H` are mutable `let` exports (constants.js), set per game via `setMapSize` from `MAP_SIZES` (S 96√ó60 ¬∑ M 128√ó80 ¬∑ L 160√ó100 default ¬∑ XL 208√ó130 ¬∑ HUGE 256√ó160, all 1.6 aspect). CELL=10px. `applyMapSize(idx)` (ui.js) resizes the canvas (dpr clamped so the backing store stays ‚â§~4000px), reallocates Flux scratch, and `resetVisuals()` drops the W√óH caches; `newGame` then allocates sim arrays and rebakes. Replays/challenges/ghosts store the size index (`rec.sz`) and reapply it ‚Äî required, since `genTerrain` consumes a size-dependent number of `rng()` calls. World-bake coarse dims (`BCW/BCH/BSW/BSH` in world.js) recompute in `ensureBake`. Fixed timestep TICK=1/30, speeds √ó1/2/4/8 (frame cap 24 at 8√ó, else 12).
- **Flux (creeper)**: Float32Array double buffer. Effective height of a cell = `ter*TER_H + depth` (TER_H=3.0, terrain levels 1‚Äì12, `TER_LEVELS` in world.js). Each tick a cell sends FLOW=0.25 of the positive height difference to each lower neighbor, total outflow capped at 0.9√ó its volume. Evaporation is near-zero (√ó0.9998 ‚àí0.0004/tick) ‚Äî **the Flux expands for the whole game and never recedes**; the extra 0.02 decay only kicks in once all emitters are dead. CREEP_MAX=50 per cell > max terrain height 36, so deep Flux can eventually crest even L12 cliffs ‚Äî high ground buys time, not safety.
- **Emitters**: emit every 0.7s; amount = `diff.amt √ó e.str √ó warmup √ó (1 + t/diff.grow)` where warmup ramps 0.3‚Üí1 over 25s. Each emitter rolls a seeded strength tier `e.str` at placement (Weak √ó0.8 25%, Standard √ó1.0 35%, Strong √ó1.5 25%, Massive √ó2.2 15%), drawn physically bigger with strength (scale `0.72+0.31¬∑str`); the hover tooltip names the tier and live output. Placed ‚â•20 cells apart on low ground, with carved basins.
- **Network**: backbones = *built, non-moving* core/collector/relay. Backbone‚Üîbackbone edge if dist ‚â§ max of their linkR. BFS from core sets `conn` + `parent` (a moving core = whole network down). Non-backbones and unbuilt ghosts attach to the nearest connected backbone within *that backbone's* linkR; moving structures never attach. Packet paths walk the parent chain and are snapshotted at dispatch.
- **Economy**: one global energy pool; production/capacity only from built + connected structures. Dispatcher: 1‚Äì4 packets per 0.1s window (burst = `1+min(3,energy/20)` ‚Äî reserves speed up logistics), 1 energy each, round-robin (`S.rr`). `S.supplyMode` ('balanced'/'weapons'/'build', P key or ‚áÑ button) sets lane order: balanced alternates construction-vs-ammo fairly via `S.dispFlip`; the other modes force one lane first. HUD SUPPLY row shows waiting counts (B build ¬∑ A ammo, via `needOf`). Packets travel 170 px/s √ó(1+buffs.pspeed); energy refunded if the target dies in flight. HUD "FLOW" = production minus 1s-windowed spend rate.
- **Damage**: Flux depth >0.05 on a footprint deals `(0.8 + 1.4¬∑min(depth,6))`/s; built structures self-repair 0.6/s when clear; ghosts spawn at 50% hp. Core death = instant loss.
- **Win**: a nullifier charges 20 packets, fires after a 1.2s beam, destroys all emitters within range 7, and is consumed. All emitters dead = win; residual Flux then decays.
- **Terp (CW3-style)**: select a built Terp ‚Üí paint target levels (1‚Äì12) on cells within `workR` 8 via `terraPaint` (click/drag; `[`/`]` or the ‚àí/+ `#terraBar` set `terraTarget` in state.js). One packet (`kind:'terra'`) = one level step via `applyTerraStep`; jobs queue on `b.tjobs`, painting a cell's current level erases its job. Cells under buildings/emitters/relics can't be painted.
- **Power Zones**: each nullified emitter leaves a zone in `S.pzones`; a building whose footprint covers one gets `b.pz` (set in `place`/`startMove`/on nullify): reactor prod √ó6, collector field √ó3, weapons √ó2 fire rate + √ó1.3 range, battery cap √ó3, backbone linkR √ó1.5 (`linkRof` in net.js).
- **Collector field economy**: `computeCoverage` (net.js, runs with every `recomputeNet`) assigns each ground cell to at most one built collector; collector income = `0.003 √ó b.cov` e/s (‚âà0.4 max), so territory is the resource. Core/reactors keep flat prod.
- **Surge tides**: `S.surge` schedules a map-wide √ó2.5 emitter surge for 8 s every 75‚Äì135 s (`tickSurge`, rescheduled with `st.rng` ‚Äî deterministic). Warning 8 s ahead: toast + alarm + music intensity forced up (updateHUD) + emitters visibly pulse (drawEmitters `surgeK`).
- **Erosion**: cells with Flux depth >6 press on higher 4-neighbors (`tickErosion`); pressure accumulates in `S.ero` and at threshold the wall cell drops one level (min 1) with a `retileTerrain` patch. Walls are temporary; Terps can rebuild them.
- **Spore Towers**: `DIFFS.spores` per map (1/2/3/5), placed mid-elevation away from emitters. From t‚âà120s each launches a spore every 40‚Äì65s at a random built structure (`tickSpores`, `st.rng`-driven): 11s arc flight, payload +9 Flux in r2.5 on impact. Beams auto-intercept spores in range. Towers are nullifier targets, leave Power Zones, and **win requires all emitters AND spore towers dead** (`checkWin`).
- **Anti-Flux**: second fluid field `S.anti`/`S.antiBuf` sharing `flowPass` physics, gated by `S._antiAny` (zero cost until a Sprayer fires). Annihilates 1:1 with Flux on contact (counted into `stats.flux`), decays faster (√ó0.999 ‚àí0.001). Rendered as pale frost in `renderCreeper`.
- **Emitter personalities** (`tickPersonalities`, assigned in `placeEmitters`): `breeder` (normal+, 70%) spawns a √ó0.8 `spawn` child every 180s (‚â§3 alive, carves a basin + `retileTerrain`); `migrant` (hard+, 60%) burrows 12‚Äì20 cells away once when a charging nullifier is within 9 cells; `boss` (INSANE only, str 3.0, `shield:1`) absorbs the first nullifier strike. Tooltips name the kind.
- **Daily Challenge**: menu button starts NORMAL with a date-derived seed (`newGame(dk,fixedSeed,opts)`) ‚Äî same map for everyone that day.
- **Orbital core respawn**: core death no longer ends the game instantly ‚Äî `killBuilding` sets `S.coreDown`/`reclaimT=t+25`, phase‚Üí`placeCore`, `coreId=0`; you redeploy a free core at a flat site (place() detects `coreDown`). The 25 s window expiring ‚Üí `gameEnd(lost)` (checked at top of `tick`). `stats.coresLost` tracked.
- **Shield** (`tickBuildings` shield branch): drains `1.5¬∑TICK` from `S.energy` each active tick, then pushes Flux from each in-range cell to its outward radial neighbor. `b.active` flag drives the render dome.
- **Converter / emitter capture**: 40-packet charge, then flags the nearest in-range emitter `e.captured` + `alive=false` (counts as neutralized for win); `tickEmitters` makes captured emitters pump Anti-Flux into `S.anti`. Single-use like the nullifier.
- **Weather** (`tickWeather`): a 10 s map-wide Flux-rain squall every ~150‚Äì250 s, scattered via `st.rng` (‚âà40 drops/tick, capped at depth 6), announced 8 s ahead (music + toast, like surges). Diagonal-streak overlay in render.
- **Magma emitter** (`kind:'magma'`, hard/insane): `tickMagma` (every 0.5 s) solidifies nearby deep Flux into permanent rock (+1 terrain, `retileTerrain`) ‚Äî the inverse of erosion.
- **Minimap** (`drawMinimap`, `MM` export): corner thumbnail (terrain cached per seed) + live Flux/network/threat blips + viewport rect; ui.js `inMM`/`mmJump` give click/drag-to-jump.
- **Records / achievements** (`storage.js`): `recordResult` on game end persists best times per difficulty + daily history + 10 achievements (`computeUnlocks` in ui.js); shown on the menu and end screen. Skipped for sandbox/replay.
- **Settings** (`state.settings`, persisted): music/sfx volume sliders (`applyVolumes`), screen-shake toggle (gates render shake), colorblind Flux tint (blue‚Üíamber in `renderCreeper`). **Show-all-ranges**: `T` toggles `showRanges` (faint ring on every tower).
- **Challenge codes**: COPY CHALLENGE encodes the finished run (seed+diff+mods+time+won) as base64; TAKE CHALLENGE plays that exact map live with `challengeTarget`, and showEnd prints a beat/lost verdict. Same `encodeRun`/`decodeRun` as replays.
- **Deferred (next wave)**: Void chasms (level-0 drain terrain ‚Äî needs careful bake/fluid/placement integration) and the Guppy (air packet-ferry ‚Äî reworks logistics). Both are deep system changes intentionally left for a dedicated verified pass rather than rushed.
- **Digitalis** (`tickDigitalis`): `S.digi` byte grid grows (hash-deterministic, 0.5s cadence) from emitters through cells with Flux >0.3, withers without Flux. Flux equalizes along the web ignoring terrain (paired exchange, conservative). **Runners** spawn on it after t>150 (rng, ‚â§8 alive), greedy-walk toward the nearest structure and stun it 6 s (`b.stun` ‚Äî stunned structures skip their whole action branch). **Snipers** one-shot runners.
- **Aether Nodes + Harvester**: 5 crystal nodes/map (`placeNodes`, mid-elevation, away from emitters/totems/relics). A node blocks normal builds; a Harvester *must* sit on one (canPlace `onNode` logic) and mines +0.08 aether/s. Rendered by `drawNodes`; minimap-blipped.
- **Minimap** is its own panel canvas (`#mini`, `miniCv`/`miniCtx`) ‚Äî `drawMinimap` renders to it, never over the playfield. `jumpCamFrac` + the `onMini*` handlers (ui.js) give click/drag-to-jump.
- **Totems + Forge**: 3 totems/map (`placeTotems`); a connected backbone within 5 cells makes one channel +0.05 aether/s into `S.aether`. The Forge panel (`#forge`, fg* buttons ‚Üí `forgeBuy(st,key)`) sells repeatable upgrades at √ó1.6 escalating cost: rate +10%, pspeed +15%, prod +0.5, **dmg +15%** (`buffs.dmg` scales cannon/mortar/strafer damage).
- **Cryo Tower**: 3 ammo per pulse (8s cd): zeroes Flux in range 5 and raises uncovered cells +2 levels as ice (`S.ice`, melts after 25 s via `tickIce`, both ends `retileTerrain`).
- **Strafer**: pad building; its aircraft (`b.air` state machine idle‚Üíout‚Üístrafe‚Üíback) sorties to the deepest Flux within 22 cells, strafes ~2.2 s (0.55/tick √ódmg over r1.5), costs 4 ammo/sortie.
- **Replay & ghost** (ui.js): every player action is recorded as `[tickN, op, args]` (`recAct`; ops P/D/M/T/S/F) alongside seed+diff+mutators; `S.tickN` increments per tick and `pumpReplay` applies queued actions before the matching tick ‚Äî bit-faithful playback. `encodeRun`/`decodeRun` = JSON‚Üîbase64 strings (COPY REPLAY / WATCH REPLAY CODE buttons). RACE GHOST restarts the same seed and overlays the previous run's placements (`state.ghost`, `drawGhost`). `replayMode` locks all mutating input.
- **Mutators** (menu checkboxes ‚Üí `S.mods`): **erosion** (default ON ‚Äî `tickErosion` only runs when set, so Flux terrain erosion is chooseable), frenzy (surges every 35‚Äì60s), fragile (hp √ó0.6), overclock (prod √ó1.5). **Sandbox** (`S.sandbox`): energy pinned at 999, no recording.
- **Relics**: 3 per map (`placeRelics`, high ground ‚â•L8, ‚â•30 cells apart): kinds rate/speed/energy. Holding a built+connected backbone within 5 cells for 12 s claims a permanent buff into `S.buffs` (fire rate +15%, packet speed +25%, +0.6 e/s) ‚Äî applied in `tickBuildings` cooldowns, `tickPackets` speed, `tickEnergy` prod. Relic cells are unbuildable ('Relic site').
- **Camera & mobile**: world drawing runs under `setTransform(dpr¬∑z, ‚Ä¶, ‚àícam¬∑z¬∑dpr)`; canvas backing store is W√óH√ódpr (dpr = min(2, ~4000/max(W,H), devicePixelRatio) so huge maps stay under GPU limits), zoom z clamped 1‚Äì6 (`ZMAX` in ui.js), pan clamped to map bounds, camera resets each `newGame`. Canvas-space text (toasts, PAUSED, placement warnings) scales by `uiScale` = logical width / CSS width (1‚Äì3) so it stays readable on phones. `canvasPos` maps client ‚Üí world through the camera; sim coordinates are untouched. Layout breakpoints: portrait ‚â§860px stacks panel below canvas; short landscape ‚â§1000√ó560 keeps a slim side panel. Touch-only CSS via `(pointer:coarse)`.
- **Game feel**: HUD has energy + threat bars (threat = fraction of cells with Flux >0.5, bar saturates at 40% coverage); core <45% hp triggers an alarm sfx + toast (4 s cooldown) and a pulsing red vignette below 60%. A hover tooltip (`drawHoverInfo` in render.js) follows the pointer: building stats, or ground level + Flux depth on open terrain (hidden in build/recycle/relocate modes). End screen shows a rank ‚Äî S: win <5 min with 0 losses, A: <7 min ‚â§2 lost, B: <10 min ‚â§5 lost, C: other wins, F: loss (`rankOf` in ui.js).

## Conventions & constraints

- Vanilla ES2017+ modules (strict by default), no frameworks, no build step. `storage.js` is the **one** deliberate localStorage user (player settings + records + achievements) ‚Äî it never feeds the sim, so determinism is untouched; everything else stays storage-free.
- In the legacy inline build (`fluxfront.html`), the literal sequence `</script` must never appear inside the JS. The legacy file is desktop-only and frozen as the sim-behavior reference; mobile/camera features live only in the modules.
- Hotkeys: **WASD / arrow keys pan the camera** (held keys tracked in `camHeld`, applied each frame by `applyCamKeys` for smooth framerate-independent pan at constant on-screen speed; works while paused; cleared on window blur), **+ / ‚àí zoom** (centred on the view via `zoomAt(f,W/2,H/2)`); 1‚Äì9,0 build (first ten), then C cryo ¬∑ N sniper ¬∑ V strafer ¬∑ B shield ¬∑ K converter ¬∑ J bomber; X recycle, G move-mode, `[`/`]` Terp level, T show-all-ranges, P supply mode, R record webm clip (`toggleRecord`, MediaRecorder + `cv.captureStream`), `?` keyboard cheat-sheet, Esc/right-click cancel, Space pause, F speed cycle, M mute, H help. `onKey` ignores keys while an INPUT/TEXTAREA/SELECT is focused. The ‚ò∞ ctrl button pauses and opens the menu mid-game (RESUME button appears; picking a difficulty starts a fresh map).
- Balance lever map: difficulty pressure = `DIFFS` (4 tiers ‚Äî easy 5/2.4/900, normal 8/3.2/650, hard 11/4.0/480, insane 14/4.8/360 as emitters/amt/grow); terrain/defense feel = `FLOW`, `TER_H`; logistics pressure = dispatcher interval (0.1s) and packet value (1e).
